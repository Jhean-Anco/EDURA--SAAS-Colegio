# Plantilla: Resumir módulo

**Cuaderno destino:** arquitectura | gestión-académica | (según módulo)
**Uso:** Obtener resumen compacto de un módulo funcional

---

## Consulta

Resumir el módulo {{NOMBRE_MODULO}} de EDURA respondiendo:

1. ¿Cuál es el propósito principal de este módulo?
2. ¿Qué actores interactúan con él y para qué?
3. ¿Cuáles son los 3-5 procesos o flujos principales?
4. ¿De qué otros módulos depende y cuáles dependen de él?
5. ¿Qué restricciones o reglas de negocio importantes aplican?

**Instrucciones:**
- Máximo 300 palabras en total.
- Citar la fuente específica para cada afirmación.
- Indicar si algún aspecto no está documentado en las fuentes cargadas.
- No inferir funcionalidades que no estén explícitas en las fuentes.
- Listar contradicciones encontradas entre documentos.
